## 一、大模型注意力机制变体总结

### 1、Multi-Head Attention 多头注意力（MHA）

Transformer 的标准注意力机制，是后续所有变体的基础。解决了早期 RNN 的上下文瓶颈问题，让每个 token 能直接关注输入序列的所有 token。

**典型模型**: GPT-2、OLMo27B、OLMo37B

**优点**: 每个注意力头独立 K/V，能捕捉不同类型的 token 关系

**缺点**: KV 缓存内存占用极高，长上下文（128k tokens）下推理效率骤降

### 2、分组查询注意力（GQA）

降低 KV 缓存开销为核心，是当前主流 LLM 的标配，中小模型的主流选择。

**典型模型**: Llama3 8B、Gemma3 27B、Sarvam3 0B

让多个 Query 头共享一组 K/V 头，减少 K/V 数量和缓存开销。内存压缩比可灵活调节（如 4 组 K/V 对应 24 个 Query 头，实现 6 倍压缩）。

### 3、多头潜在注意力（MLA）

适用于大参数模型（≥100B），在降内存的同时能保持甚至提升性能。

**典型模型**: DeepSeekV3、KimiK2、GLM-5、Sarvam 105B

核心思想是将 K/V 缓存为压缩的潜在表示，推理时再重构——本质是缓存压缩策略。

### 4、滑动窗口注意力（SWA）

限制每个 token 仅关注最近的固定窗口内的 token，同时搭配少量全局注意力层保证长距离信息传递。

**典型模型**: Gemma3 27B、OLMo3 32B、Xiaomi MiMo-V2-Flash

关键调参为局部/全局层比例（如 Gemma3 是 5:1）和窗口大小，比例越极端、窗口越小，效率越高。

### 5、DeepSeek 稀疏注意力（DSA）

是滑动窗口注意力（SWA）的进阶版，解决了 SWA 硬编码局部、无法捕捉远距离关键信息的痛点。

**典型模型**: DeepSeekV3.2、GLM-5

通过 Flash 索引器 + Top-k 选择器让模型自适应选择高相关的历史 token，形成稀疏注意力掩码。支持超长上下文（DeepSeekV3.2 支持 128k tokens）。

### 6、门控注意力（Gated Attention）

对传统全注意力层的精细化修改，增加三个关键设计：
- **输出门**：注意力结果输出前，通过门控系数缩放控制信息传递
- **QK-Norm**：用零中心的归一化替代标准 RMSNorm，提升稳定性
- **部分 RoPE**：对位置编码做轻量化处理

主要用于混合架构中的全注意力层，提升其在长上下文下的稳定性。在 **Qwen3-Next/Qwen3.5** 中应用。

## 二、Agent 工作流优化综述

Agent 的执行过程是一套工作流程（先查资料、再调用工具、再验证结果、最后出答案）。这套流程的设计好不好，比单靠 LLM 本身更影响做事的效果和成本。

### 核心框架：智能计算图（ACG）

把 LLM 的工作流程看成一张**智能计算图（ACG）**，不管简单步骤流水线还是多智能体协作，都抽象成节点 + 连线的图：

- **节点**: 原子操作（LLM 调用、检索、工具使用、验证等）
- **连线**: 动作之间的先后/依赖关系

### 两种优化方式

**1. 静态优化**

部署前就设计好固定的流程模板，后续所有任务都按这个来。优点是好调试、成本可控，缺点是灵活性差。

**2. 动态优化**

不搞固定模板，根据问题难度、执行情况临时改流程：
- **剪枝/选择**：有大的通用流程库，简单问题挑精简的步骤用
- **事前生成**：看到具体问题后，先专门为这个问题设计一套流程，再执行

### 四大评估维度

1. **有效性**: 任务准确率、成功率等
2. **效率**: token 使用、工具调用数、时间、货币成本等
3. **鲁棒性**: 扰动/工具漂移/未见过工具下的性能
4. **图级属性**: 节点数、深度、通信量、结构方差、编辑次数等
