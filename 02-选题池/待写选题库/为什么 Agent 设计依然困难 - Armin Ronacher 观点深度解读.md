# 为什么 Agent 设计依然困难 - Armin Ronacher 观点深度解读

## 选题背景

Flask 创始人 Armin Ronacher 提出：

> Agent 设计依然困难

作为资深开发者，他的观点值得深入分析。

## 核心问题

### 1. 困难在哪里
- 状态管理的复杂性
- 工具调用的可靠性
- 错误处理与恢复
- 人机协作的边界

### 2. 为什么比预期难
- 初期期望过高
- 技术成熟度不足
- 场景复杂度被低估

### 3. 解决方向
- 架构设计改进
- 工程实践优化
- 用户预期管理

## 核心观点

> Agent 不是调用 API 那么简单，是系统工程

> 好的 Agent 设计，需要承认困难而不是回避
