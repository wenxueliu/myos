# AI+ 浏览器自动化工作流设计

## 选题背景

传统自动化脚本的痛点：
- 元素定位脆弱，页面一变就失效
- 错误处理能力弱，需要人工干预
- 无法应对动态内容和复杂逻辑

AI 辅助的自动化工作流可以解决这些问题。

## 核心架构

```
┌─────────────────┐
│  OpenClaw CLI   │  ← AI 指令解析
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│    Gateway      │  ← WebSocket 通信
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│ Chrome Extension│  ← CDP 协议 + AI 元素识别
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Chrome Browser │
└─────────────────┘
```

## 核心观点

> AI 辅助的核心价值：自动识别页面元素，减少手动定位。

> 协议稳定 - 基于 CDP，不依赖易变的 DOM 结构。

## 工作流设计原则

1. **元素定位**：优先使用 AI 识别的 refs， fallback 到 ARIA
2. **错误处理**：每一步都有等待和验证
3. **状态管理**：保存登录状态，避免重复登录
4. **日志记录**：每一步操作都有日志，方便调试

## 典型工作流

### 社交媒体自动发布
1. 加载保存的登录状态
2. 打开目标平台
3. AI 识别发布按钮
4. 填写内容 + 上传素材
5. 点击发布 + 验证结果

### 电商数据采集
1. 打开商品列表页
2. AI 识别商品卡片
3. 提取数据 + 保存 JSON
4. 自动翻页 + 循环采集

## 待验证要点

1. AI 元素识别的准确率
2. 复杂页面（如 SPA）的处理能力
3. 与传统方案的稳定性对比
