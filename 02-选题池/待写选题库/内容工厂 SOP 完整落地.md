# 内容工厂 SOP 完整落地

## 选题背景

很多人用大模型效率没有质变：
- 每次都在重复无效劳动
- 没有知识沉淀
- 没有复用已验证的内容结构

系统化创作需要标准化的 7 阶段流程。

## 核心目录结构

| 目录 | 用途 |
|------|------|
| `01-灵感与素材库` | 日常灵感剪报 + 爆款素材片段 |
| `02-选题池` | 人工确认或 AI 衍生的待写命题 |
| `03-内容工厂` | 大纲挑选区、初稿打磨区、终稿确认区 |
| `04-已发布归档` | 成品最终沉淀 |

## 7 阶段标准流程

### 阶段 1：灵感捕获
- 触发：用户发送链接或原文
- 动作：整理保存 + 提炼选题

### 阶段 2：素材沉淀
- 触发：用户发送金句或逻辑框架
- 动作：打标签存入爆款素材库

### 阶段 3：选题立项与大纲生成
- 触发：用户要求开始写某选题
- 动作：检索灵感库→生成 3 个大纲→等待确认

### 阶段 4：初稿打磨
- 触发：用户确认大纲
- 动作：检索金句润色→生成初稿

### 阶段 5-7：终稿确认与归档
- 触发：用户确认终稿并已发布
- 动作：移动文件 + 更新 YAML 状态

## 强制元数据规范

每篇文档顶部必须携带标准 YAML 属性：

```yaml
---
标题：xxx
阶段：灵感/选题/大纲/初稿/终稿/已发布
关联参考：[[xxx]]
发布平台：公众号
---
```

> 这是 AI 识别文件状态的唯一索引标识。
