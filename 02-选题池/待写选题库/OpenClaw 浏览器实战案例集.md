# OpenClaw 浏览器实战案例集

## 选题背景

很多人看完教程还是不会写自动化脚本：
- 不知道从哪里开始
- 没有可复用的代码模板
- 遇到问题不知道怎么调试

需要一篇"抄作业"式的案例集，直接提供可用的代码模板。

## 核心案例清单

### 案例 1：自动登录脚本
- 场景：需要登录的自动化任务
- 核心命令：open + snapshot + type + click
- 关键点：等待页面加载、验证登录状态

### 案例 2：数据采集脚本
- 场景：电商价格监控、竞品信息抓取
- 核心命令：evaluate + JSON 输出
- 关键点：CSS 选择器定位、数据格式化

### 案例 3：表单批量提交
- 场景：批量注册、信息录入
- 核心命令：循环读取 CSV + 表单操作
- 关键点：错误处理、日志记录

### 案例 4：定时任务部署
- 场景：每日数据同步、定时发布
- 核心命令：crontab 配置
- 关键点：环境变量、错误通知

## 核心观点

> 好的自动化脚本 = 清晰的流程 + 稳定的元素定位 + 完善的错误处理

> 短期脚本用角色定位（开发快），长期维护用 ARIA 定位（更稳定）。

## 代码模板结构

每个案例包含：
1. 场景描述
2. 完整代码
3. 使用说明
4. 常见问题

## 待验证要点

1. 代码在真实环境中的可执行性
2. 是否需要补充错误处理逻辑
3. 是否增加更多场景（如文件上传、弹窗处理）
