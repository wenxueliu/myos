# 外部记忆系统的三种设计模式 - 向量数据库、知识库、反思总结

## 选题背景

要解决 LLM"跨会话遗忘"问题，必须在 LLM 之外构建独立的记忆系统。

业界已衍生出三种成熟的设计模式：
- 向量数据库模式（最流行）
- 外部知识库模式
- 反思与总结模式

需要一篇设计模式对比与选型指南。

## 三种设计模式对比

| 模式 | 存储方式 | 检索方式 | 适用场景 |
|------|----------|----------|----------|
| **向量数据库** | 向量嵌入 | 相似度搜索 | 海量非结构化记忆 |
| **外部知识库** | 结构化/半结构化数据库 | 数据库查询 | 核心实体与状态 |
| **反思总结** | 对话摘要 | 直接使用摘要 | 提炼高价值记忆 |

## 模式详解

### 1. 向量数据库模式

**代表**：Pinecone、Weaviate

**流程**：
1. 将信息通过嵌入模型转化为向量
2. 存储到向量数据库
3. 新对话时，将查询转化为向量进行相似度搜索
4. 找出最相关片段，"回填"给 LLM

> 这种方法将记忆从"死记硬背"变成了"智能检索"，相当于给 AI 装了一个可按内容寻址的"外部硬盘"。

### 2. 外部知识库模式

**代表**：PostgreSQL、MongoDB

**MongoDB 观点**：
> AI Agent 的记忆数据格式复杂多样，包含结构化的用户信息、半结构化的对话历史和完全非结构化的摘要，文档数据库的灵活模式非常适合这种场景。

**优势**：
- 支持复杂查询（如"找出用户上周所有关于退款的对话"）
- 事务一致性
- 跨 Agent 实例的记忆共享

### 3. 反思与总结模式

**核心思想**：
- 让 LLM 定期对交互进行"复盘"和"摘要"
- 将精炼的摘要存入长期记忆

> 这模仿了人类将短期经历转化为长期记忆的"消化"过程，能显著提升记忆的效率和对话的深度。

### 混合模式（实际应用中更常见）

```
┌─────────────────────────────────────────┐
│           混合记忆架构                   │
├─────────────────────────────────────────┤
│  向量数据库：海量非结构化记忆 + 语义检索  │
│  关系型数据库：结构化用户画像 + 元数据    │
│  总结任务：定期生成对话摘要              │
└─────────────────────────────────────────┘
```

## 核心观点

> 在实际应用中，混合模式更为常见。

> 这种分层、分类型的记忆架构，旨在为不同场景和数据类型提供最合适的存储与检索方案。
