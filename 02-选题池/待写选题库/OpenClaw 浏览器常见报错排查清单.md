# OpenClaw 浏览器常见报错排查清单

## 选题背景

腾讯云那篇教程的评论区暴露了大量真实问题：
- 配置成功了但机器人还说不能用
- 浏览器能启动但无法访问网络
- 分辨率很低看不清

需要一篇排查清单，按症状快速定位问题。

## 高频问题清单

### 1. which google-chrome 找不到
**原因**：Chrome 未安装或安装失败  
**解决**：重新执行安装步骤，确认 yum localinstall 成功

### 2. 配置后飞书仍提示无法使用浏览器
**原因**：配置文件中 profile 为 "messaging"  
**解决**：将 profile 改成 "full"

### 3. 浏览器分辨率很低
**原因**：viewport 未配置  
**解决**：调整 viewport 设置（待补充具体命令）

### 4. 无法访问网络
**原因**：服务器防火墙或网络限制  
**解决**：检查安全组、防火墙规则

### 5. 登录后问还是说无法使用
**原因**：Gateway 未重启或浏览器未启动  
**解决**：执行 `openclaw gateway restart` + `openclaw browser start`

## 核心观点

> 排查的本质是按顺序验证：安装 → 配置 → 重启 → 验证

> 评论区是最真实的需求来源，一篇教程的评论区就是一份故障排查手册。
