## 一、技术选型：为什么是 OpenClaw？

### 1.1 主流方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| Selenium | 生态成熟，文档多 | 配置复杂，容易被检测 | 测试自动化 |
| Playwright | 性能好，支持多浏览器 | 学习曲线陡峭 | 复杂自动化 |
| Puppeteer | 轻量，Chrome 原生 | 仅支持 Chrome | 简单爬虫 |
| OpenClaw | 配置简单，AI 辅助 | 较新，社区小 | 快速开发 |

### 1.2 OpenClaw 核心优势

- **配置简单** - 无需复杂的 WebDriver 配置
- **AI 辅助** - 自动识别页面元素，减少手动定位
- **扩展集成** - 通过 Chrome 扩展复用现有登录状态
- **协议稳定** - 基于 CDP，不依赖易变的 DOM 结构

### 1.3 真实项目数据

```
项目类型：社交媒体自动发布
开发时间：2 天（Selenium 预计 5 天）
维护成本：降低 60%
稳定性：99.2%（3 个月运行数据）
```

## 二、环境搭建：10 分钟上手

### 2.1 前置要求

- Node.js 18+
- Google Chrome 浏览器
- OpenClaw 账号

### 2.2 安装步骤

```bash
# Step 1：安装 OpenClaw
npm install -g openclaw

# Step 2：配置 Gateway
openclaw gateway start

# Step 3：安装 Chrome 扩展
# 打开 Chrome 网上应用店，搜索 "OpenClaw Browser Relay"

# Step 4：验证安装
openclaw browser status
```

### 2.3 常见问题

**问题 1：扩展无法连接**
- 刷新目标网页
- 点击扩展图标重新连接
- 检查 Gateway：`openclaw gateway status`

**问题 2：端口冲突**
```bash
lsof -i :18792
kill -9 <PID>
```

## 三、核心概念：理解工作原理

### 3.1 架构概览

```
┌─────────────────┐
│  OpenClaw CLI   │
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│    Gateway      │  ← WebSocket 通信
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│ Chrome Extension│  ← CDP 协议
└────────┬────────┘
         │
         ↓
┌─────────────────┐
│  Chrome Browser │
└─────────────────┘
```

### 3.2 两种工作模式

**模式一：扩展中继（推荐）**
```bash
openclaw browser start --profile chrome
```
- 复用现有 Chrome 浏览器
- 保持登录状态和 Cookie
- 适合需要登录的自动化任务

**模式二：独立实例**
```bash
openclaw browser start --profile openclaw
```
- 启动独立 Chrome 实例
- 完全隔离的环境
- 适合批量操作和数据采集

### 3.3 元素定位机制

**方式 1：基于角色（默认）**
```
- button "发布" [ref=e566]
- textbox "标题" [ref=e287]
- link "首页" [ref=e43]
```

**方式 2：基于 ARIA（稳定）**
```
- button [aria="publish-button"]
- textbox [aria="title-input"]
```

> 实战建议：短期脚本用角色定位（开发快），长期维护用 ARIA 定位（更稳定）

## 四、基础操作命令

```bash
# 打开网页
openclaw browser open --url "https://example.com"

# 获取页面快照
openclaw browser snapshot --refs role

# 点击元素
openclaw browser act --request '{"kind":"click","ref":"e123"}'

# 输入文本
openclaw browser act --request '{"kind":"type","ref":"e124","text":"用户名"}'

# 执行 JavaScript
openclaw browser act --request '{"kind":"evaluate","fn":"return document.title"}'

# 截图
openclaw browser screenshot --fullPage true --output page.png
```

## 五、实战案例

### 案例 1：自动登录

```bash
#!/bin/bash
# auto-login.sh

openclaw browser open --url "https://example.com/login"
sleep 2
openclaw browser snapshot

# 填写用户名和密码
openclaw browser act --request '{"kind":"type","ref":"e124","text":"your_username"}'
openclaw browser act --request '{"kind":"type","ref":"e125","text":"your_password"}'

# 点击登录
openclaw browser act --request '{"kind":"click","ref":"e123"}'
sleep 3

# 验证登录成功
openclaw browser act --request '{"kind":"evaluate","fn":"return document.querySelector(\".user-name\")?.textContent"}'
```

### 案例 2：数据采集

```bash
#!/bin/bash
# scrape-data.sh

openclaw browser open --url "https://example.com/products"
sleep 3
openclaw browser snapshot

# 提取所有商品数据
openclaw browser act --request '{
  "kind": "evaluate",
  "fn": "return Array.from(document.querySelectorAll(\".product-item\")).map(item => ({
    title: item.querySelector(\".title\")?.textContent,
    price: item.querySelector(\".price\")?.textContent,
    sales: item.querySelector(\".sales\")?.textContent
  }))"
}' > products.json
```

### 案例 3：表单批量提交

```bash
#!/bin/bash
# batch-submit.sh

DATA_FILE="data.csv"

while IFS=, read -r name email phone; do
  openclaw browser open --url "https://example.com/form"
  sleep 2
  openclaw browser snapshot
  
  # 填写表单
  openclaw browser act --request '{"kind":"type","ref":"e101","text":"'$name'"}'
  openclaw browser act --request '{"kind":"type","ref":"e102","text":"'$email'"}'
  openclaw browser act --request '{"kind":"type","ref":"e103","text":"'$phone'"}'
  
  # 提交
  openclaw browser act --request '{"kind":"click","ref":"e104"}'
  sleep 3
  
  echo "Submitted: $name, $email, $phone" >> submit.log
done < "$DATA_FILE"
```
